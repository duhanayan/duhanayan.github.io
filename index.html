Stack Clone

Unity ile geliştirilen minimal ama zorlu bir mobil beceri oyunu

Oyun Hakkında

Stack, Ketchapp'in popüler mobil oyunlarından biridir. Bu proje, oyunun temel mekaniğini sıfırdan tasarlayarak kendi yorumumla yeniden inşa ettiğim bir clone çalışmasıdır. Amaç: teknik kabiliyetlerimi pekiştirmek ve performans odaklı sistem mimarisi inşa etmektir.

Ekran Görüntüleri

Öne Çıkan Özellikler

Performans

  • Object Pooling ile sıfıra yakın Instantiate
  • Manuel bellek yönetimi (GarbageCollector)
  • Maksimum 60 FPS hedefi

Modüler Mimari

  • Singleton servis katmanı (Audio, UI, Score)
  • Olay tabanlı iletişim
  • Temiz ayrıştırılmış sorumluluklar

Görsel Dinamizm

  • Uyumlu background renk geçişleri
  • Slice bazlı kesme & animasyon
  • Sarsıntısız kamera takibi

Esnek Kod Yapısı

  • Yüksek okunabilirlik, Clean Code prensipleri
  • Extensibility için arayüzler
  • Unit test'e uygun metot tasarımı

Kullanılan Araçlar

DOTween

Animasyon işlemleri (UI fade, hareket geçişleri, zamanlayıcılar) için kullanıldı. Performanslı tween işlemleri ile daha yumuşak bir kullanıcı deneyimi sağlandı.

Git

Versiyon kontrolü ve proje yönetimi için Git kullanıldı. Feature-branch yapısıyla çalışıldı, kod bütünlüğü korundu.

Tasarım Desenleri & Mimari

Projenin omurgasını; Singleton (servis yöneticileri), Object Pool (SlicePool), ve sınırlı Service Locator yapısı oluşturmaktadır. Bu sayede:

  • GC baskısı minimuma indirilir.
  • Bileşenler arası gevşek bağlılık sağlanır.
  • Yeni özellik eklemek daha az refactor gerektirir.

Karşılaşılan Zorluklar & Çözümler

FPS Düşüşleri

Gerçek zamanlı kesme & düşen slice instantiate işlemleri FPS'i etkiliyordu. Object Pooling ile sahnede yeniden kullanım sağlanarak CPU spike'ları %70 azaltıldı.

Renk Senkronizasyonu

Arka plan ve slice renklerinin uyumsuz geçişlerinden kaçınmak için BackgroundColor sınıfı HSV interpolasyonu kullanıyor.

Kamera Takibi

Kamera CameraController scripti ile dinamik hedef yüksekliğine yumuşak geçiş animasyonu uygular, böylece 'snap' hissi ortadan kalkar.
